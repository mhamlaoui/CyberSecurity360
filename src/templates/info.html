INFORMATIONS

DDoS (Distributed Denial of Service)

Définition

Une attaque en déni de service ou en déni de service distribué (DDoS pour Distributed Denial of Service en anglais) vise à rendre inaccessible un serveur par l'envoi de multiples requêtes jusqu'à le saturer ou par l'exploitation d'une faille de sécurité afin de provoquer une panne ou un fonctionnement fortement dégradé du service.

Comment ça fonctionne ?

Les attaquants créent un botnet en infectant des appareils connectés (PC, IoT, routeurs…) avec des malwares. Ce réseau, contrôlé à distance, sert à lancer des attaques DDoS en inondant une cible de trafic.

Types d'attaques DDoS

  • Attaques volumétriques : saturent la bande passante avec un très grand volume de trafic
  • Attaques de protocole : exploitent des failles des protocoles réseau pour épuiser les ressources
  • Attaques applicatives (couche 7) : ciblent des applications web avec des requêtes légitimes en apparence

Phishing

Définition

Le terme phishing fait référence aux tentatives de vol d'informations sensibles, telles que des noms d'utilisateur, mots de passe, numéros de carte bancaire ou d'autres données confidentielles, dans le but de les utiliser ou de les vendre.

Comment ça fonctionne ?

Dans une attaque typique, le criminel collecte les coordonnées d'une ou plusieurs cibles, puis envoie des messages de phishing par email ou SMS.

Types d'attaques phishing

  • Spear phishing : attaque ciblée sur une personne précise
  • Whaling : forme de spear phishing visant des dirigeants
  • BEC (Business Email Compromise) : usurpation d'un cadre pour détourner des paiements

SQL Injection

Définition

Les attaques par injection SQL exploitent les failles de sécurité d'une application qui interagit avec des bases de données.

Comment ça fonctionne ?

Une application web insère directement des données saisies par l'utilisateur dans une requête SQL sans vérification ni filtrage.

Types d'attaques SQL Injection

  • SQLi en bande : exploite les messages d'erreur de la base
  • SQLi aveugle : déduit les infos selon si la réponse change ou non
  • SQLi hors bande : exploite des fonctions du serveur pour exfiltrer les données

Ransomware

Définition

Les rançongiciels (ou ransomwares) sont des logiciels malveillants qui bloquent l'accès à un ordinateur ou à des fichiers en les chiffrant, puis réclament à la victime le paiement d'une rançon pour en retrouver l'accès.

Comment ça fonctionne ?

L'infection peut survenir après l'ouverture d'une pièce jointe piégée, en cliquant sur un lien malveillant, en naviguant sur un site compromis, ou suite à une intrusion dans le système.

Types d'attaques par ransomware

  • Chiffrement : les fichiers sont chiffrés et une rançon est exigée
  • Verrouillage : l'accès à l'ordinateur est complètement bloqué
  • Extorsion : l'attaquant menace de divulguer les données volées

Malware

Définition

Un malware (ou logiciel malveillant) est un programme conçu pour infiltrer, perturber ou endommager un ordinateur ou un réseau, généralement à l'insu du propriétaire.

Types d'attaques par malware

  • Virus : se propage via des fichiers et infecte d'autres programmes
  • Vers (Worms) : se propagent automatiquement sur un réseau
  • Chevaux de Troie (Trojans) : se présentent comme des logiciels légitimes

Man-in-the-Middle (MITM)

Définition

Une attaque Man-in-the-Middle (MITM) se produit lorsqu'un attaquant s'interpose secrètement entre deux parties qui communiquent, dans le but d'intercepter, d'espionner, d'altérer ou de rediriger leurs échanges de données.

Types d'attaques MITM

  • Interception de communication : intercepte les données échangées
  • Injection de contenu : modifie le contenu transmis
  • Attaque de redirection : redirige vers un faux site Web

Rapport d'Analyse des Menaces de Cybersécurité (2015–2024)

Résumé

Ce projet présente une analyse des menaces de cybersécurité mondiales sur une période de 10 ans, basée sur des incidents de cybersécurité répertoriés dans différents pays, secteurs et types d'attaques. L'objectif est d'identifier les tendances, les vulnérabilités prédominantes et les stratégies de défense les plus efficaces face à l'évolution des menaces.

Source des données

Les données utilisées dans ce projet proviennent d'une base de données accessible sur Kaggle, une plateforme de données bien connue. Cette base contient un ensemble d'incidents de cybersécurité répertoriés sur une période de 10 ans (2015–2024).

Description de la base de données

  • Pays : Localisation géographique de l'incident
  • Année : Période temporelle de l'incident (2015–2024)
  • Type d'attaque : Méthodologie utilisée par les attaquants
  • Secteur ciblé : Secteur économique affecté
  • Pertes financières : Impact économique en millions de dollars
  • Nombre d'utilisateurs affectés : Impact sur les individus

Aperçu des Tendances Principales

Distribution géographique des incidents

Les attaques sont concentrées dans des pays technologiquement avancés, tels que la Chine, les États-Unis, le Royaume-Uni, l'Allemagne et l'Inde.

Évolution temporelle (2015–2024)

  • 2015–2017 : Attaques principalement basées sur l'hameçonnage
  • 2018–2020 : Augmentation des rançongiciels
  • 2021–2024 : Apparition d'attaques plus sophistiquées

Industries ciblées

  • Secteur bancaire et financier : Cible principale
  • Gouvernement : Cible d'attaques sophistiquées
  • Éducation : Ciblée par des attaques de phishing
  • Télécommunications : Souvent visées par des attaques DDoS

Sources des attaques

  • Groupes de hackers : Motivés par des gains financiers
  • États-nations : Attaques d'espionnage ou de sabotage
  • Menaces internes : Risque important, difficile à détecter

Mécanismes de défense

  • Réseaux privés virtuels (VPN)
  • Pare-feu
  • Solutions antivirus
  • Détection basée sur l'IA

Analyse des Impacts

Impact financier

Les pertes variant de quelques dizaines à plusieurs centaines de millions de dollars. Les secteurs bancaire, financier et de la santé sont les plus affectés.

Impact sur les utilisateurs

Les incidents touchent des millions d'utilisateurs, avec des conséquences graves lorsque des données sensibles sont compromises.
